CryptoRadar: Clean Architecture ile Modern Android Kripto Para Takip Uygulaması

Kotlin · Jetpack Compose · Clean Architecture · MVVM · Coroutines & Flow · Ktor · Koin · Material Design 3 · Kotlin Serialization

GitHub
Kısa Özet
Bu projede, Clean Architecture prensiplerini kullanarak Android için bir kripto para takip uygulaması geliştirdim. Coincap API'den gerçek zamanlı veri çekerek kullanıcılara kripto para fiyatlarını, piyasa değerlerini ve geçmiş fiyat grafiklerini sunuyorum. Uygulama, Material Design 3 Adaptive Navigation kullanarak hem telefon hem tablet için optimize edilmiş bir kullanıcı deneyimi sunuyor.
Projenin Amacı
Bu çalışmayı gerçekleştirirken temel hedefim, Clean Architecture'ın Android geliştirmede nasıl uygulanacağını ve katmanlar arası bağımlılıkların nasıl minimize edileceğini göstermekti.

Genellikle Android projelerinde UI ve business logic iç içe geçer, bu da test edilebilirliği ve bakımı zorlaştırır. Ben bu projede Domain katmanını tamamen bağımsız hale getirerek, Data katmanını (API implementasyonu) ve Presentation katmanını (UI) birbirinden izole ettim. Böylece API değiştiğinde sadece Data katmanını, UI değiştiğinde sadece Presentation katmanını güncellemek yeterli oldu. Ayrıca Result pattern ile tip-güvenli hata yönetimi, safeCall wrapper'ı ile merkezi exception handling ve özel Canvas tabanlı LineChart bileşeni ile interaktif grafik görselleştirme gibi modern Android geliştirme tekniklerini uyguladım.
Mimari & Yapı
Uygulama Clean Architecture prensiplerine uygun olarak Data, Domain ve Presentation katmanlarına ayrıldı.

Domain: Business logic ve domain modelleri (Coin, CoinPrice) ile CoinDataSource interface'i. Bu katman hiçbir Android bağımlılığı içermez ve tamamen platform bağımsızdır.

Data: API implementasyonları (RemoteCoinDataSource), DTO'lar (CoinDto, CoinHistoryDto), mapper fonksiyonları ve HTTP client konfigürasyonu. Ktor ile network katmanı, Kotlin Serialization ile JSON parsing ve Result pattern ile tip-güvenli hata yönetimi burada gerçekleşir.

Presentation: Jetpack Compose UI bileşenleri, ViewModel (StateFlow ile reactive state management), Material Design 3 Adaptive Navigation ve özel Canvas tabanlı LineChart bileşeni. Koin ile dependency injection kullanarak ViewModel'leri ve data source'ları yönetiyorum.

Ek olarak, core modülünde tüm feature'ların kullanabileceği ortak utility'ler (Result pattern, NetworkError handling, safeCall wrapper) ve navigation yapısı bulunuyor. Bu sayede kod tekrarı önleniyor ve proje genişletilebilir bir yapıya sahip oluyor.
