USG Challenge Mobil Uygulama: Clean Architecture ile Türkiye Şehir Rehberi

Kotlin · Jetpack Compose · Clean Architecture · MVVM · Koin · Ktor · Room · Google Maps API · Coroutines · Flow

GitHub
Kısa Özet
Bu projede, Clean Architecture prensiplerine uygun bir Android uygulaması geliştirdim. Türkiye'deki şehirleri ve konumları listeleyen, favorilere ekleyen ve haritada görüntüleyen, kullanıcı odaklı bir şehir rehberi uygulaması oluşturdum. Uygulama, feature-based organizasyon, MVVM deseni ve modern Android teknolojileriyle geliştirildi.
Projenin Amacı
Hedef, Clean Architecture'ın iş kurallarına uygun ve sürdürülebilir bir Android uygulaması geliştirmekti. Business logic ile UI katmanlarını izole ettim. Feature-based yapıyla her özelliği bağımsız modüller olarak organize ettim. Böylece yeni özellikler eklemek ve bakım yapmak daha kolay oldu.

Koin ile bağımlılık yönetimi, Ktor ile ağ katmanı, Room ile yerel veritabanı yönetimi, Flow ile reaktif state yönetimi ve Jetpack Compose ile modern UI oluşturdum. Bu yapı sayesinde test edilebilir, bakımı kolay ve ölçeklenebilir bir mimari kurdum.
Mimari & Yapı
Uygulama, Clean Architecture prensiplerine uygun olarak Data, Domain ve Presentation katmanlarına ayrıldı. Her feature (Splash, Home, Detail, Location Map, Favorites) kendi içinde bu üç katmanı barındırır.

Data Layer (Veri Katmanı): API entegrasyonları için Ktor client kullanımı Room veritabanı ile yerel favori yönetimi Repository implementasyonları ve DAO'lar Entity-to-Domain model mapper'ları

Domain Layer (Alan Katmanı): Use case'ler (GetCitiesPageUseCase, GetLocationDetailUseCase, ToggleFavoriteUseCase, vb.) Repository interface'leri (soyutlama katmanı) Domain modelleri ve business logic

Presentation Layer (Sunum Katmanı): MVVM deseni ile ViewModel'ler Jetpack Compose ile UI bileşenleri Action-State pattern ile kullanıcı etkileşimleri Flow ile reaktif state yönetimi Navigation Manager ile merkezi navigasyon yönetimi
